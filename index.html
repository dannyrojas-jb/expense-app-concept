How this concept follows the documented process

Step 1FSOs submit payment requestsBulk document upload, then one data collection screen per document: payee, date, cost split across funding sources, description. Try it on the Payment Request page.
Step 2Admin reviews with per-field verificationHalf-screen document, half-screen fields. Approve needs every checkbox ticked; Return sends field comments back to the FSO in red; Reject sends a message.
Step 3Approved expenses go to the bookkeeperA running report of expenses to add to QuickBooks (simulated here), plus received income per project.
Step 4Tracking per project by budget line itemFunding overview, spenddown by line item, filterable expense lists, and PDF / Excel report generation on the Tracking page.
